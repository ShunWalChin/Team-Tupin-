## 3. Acessos iniciais

As senhas nao devem ficar escritas neste README. Guarde em um gerenciador de senhas ou em um local seguro, compartilhado somente com quem precisa operar o sistema.

| Perfil | Usuario | Uso |
| --- | --- | --- |
| Admin | `teamadmin` | Gestao geral do sistema |
| Professor | `professor.teste` | Testar visao de professor |
| Aluno | `aluno.teste` | Testar visao de aluno |

Boas praticas:

- Trocar senhas temporarias antes de usar com alunos reais.
- Nao enviar senha em grupo de WhatsApp.
- Nao reutilizar essas senhas em Instagram, email, Hotmart ou qualquer outro servico.
- Se alguem sair da equipe, trocar a senha imediatamente.

## 4. Rotina diaria da Isa

1. Abrir o app: https://app.teamtupina.com.br/pt-br/user/login
2. Entrar como admin.
3. Conferir se ha alunos novos, alunos em acompanhamento ou pendencias.
4. Validar se professores conseguem acessar seus alunos e rotinas.
5. Conferir se os CTAs do site continuam levando para WhatsApp, RESET 7 ou app.
6. Testar rapidamente o site no celular pelo menos uma vez por semana.

O foco diario e garantir que a experiencia esteja funcionando para tres pessoas:

- Quem acabou de conhecer a marca.
- Quem comprou/entrou agora.
- Quem ja e aluno e precisa de constancia.

## 5. Fluxo de novo aluno

Use este fluxo sempre que uma pessoa entrar na consultoria:

1. Receber o contato no WhatsApp ou outro canal oficial.
2. Confirmar nome, email, telefone, objetivo e observacoes importantes.
3. Criar ou revisar o cadastro no app.
4. Definir se o aluno sera acompanhado diretamente pela Isa ou por professor.
5. Garantir que o aluno recebeu:
   - link de acesso;
   - usuario;
   - senha inicial segura;
   - orientacao do que fazer no primeiro acesso.
6. Conferir se o treino/plano inicial foi cadastrado.
7. Fazer uma mensagem curta de boas-vindas explicando o proximo passo.

Mensagem base:

```text
Oi! Seu acesso ao Team Tupina esta pronto.

Entre pelo link do app, faca login e confira as primeiras orientacoes.
Qualquer duvida no primeiro acesso, me chama aqui que eu te ajudo.
```

## 6. Fluxo de professor

O professor usa o sistema para acompanhar alunos, rotinas e progresso.

Antes de liberar um professor real:

1. Criar o usuario do professor.
2. Garantir que ele nao esta como admin root.
3. Vincular ao grupo/perfil de professor.
4. Fazer um teste de login.
5. Pedir para ele revisar o app em celular e computador.
6. Confirmar que ele enxerga apenas o que precisa para trabalhar.

Checklist do professor:

- Consegue fazer login.
- Consegue acessar area de treinos.
- Consegue visualizar alunos atribuídos.
- Consegue criar ou revisar planos.
- Consegue acompanhar progresso sem depender do admin.

## 7. Fluxo de aluno

O aluno precisa de uma experiencia simples.

No primeiro acesso, ele deve saber:

- onde entrar;
- qual usuario usar;
- qual e a senha inicial;
- onde ver o treino;
- como avisar se algo nao estiver funcionando;
- como registrar progresso, se isso fizer parte do acompanhamento.

Checklist do aluno:

- Login funciona.
- Dashboard abre.
- Treino ou rotina aparece.
- Informacoes principais estao claras.
- Ele sabe quem chamar em caso de duvida.

## 8. RESET 7

O RESET 7 e o produto de entrada do ecossistema.

Objetivo:

- gerar primeira compra;
- entregar valor rapido;
- aproximar a pessoa da consultoria;
- abrir conversa para produtos maiores.

Onde aparece:

- Site: https://teamtupina.com.br/#ebooks-caros
- Materiais internos: arquivos `RESET7_*` na pasta do projeto.

Rotina recomendada:

1. Conferir se a capa do RESET 7 esta correta.
2. Conferir se o preco e a promessa estao atualizados.
3. Validar o link de compra antes de divulgar.
4. Manter as mensagens de venda alinhadas com o produto real.
5. Depois da compra, convidar a pessoa para uma conversa sobre consultoria.

## 9. Site institucional

O site deve ser tratado como vitrine oficial.

Antes de pedir alteracoes, defina:

- qual dobra precisa mudar;
- qual texto entra;
- qual imagem sera usada;
- qual link o botao deve abrir;
- se a mudanca e urgente ou pode esperar uma revisao visual.

Principais areas do site:

- Hero inicial.
- Metodo.
- Equipe.
- Jornada.
- Prova social.
- RESET 7.
- Contato.

Cuidados:

- Nao deixar textos longos demais em cards.
- Nao trocar imagem sem revisar no celular.
- Nao criar muitos botoes competindo entre si.
- Sempre testar o link do WhatsApp depois de alterar CTA.

## 10. App wger personalizado

O app e baseado no wger, personalizado para o Team Tupina com tecnologia F.A.T Tech.

Ele pode cuidar de:

- usuarios;
- professores;
- alunos;
- treinos;
- exercicios;
- medidas;
- progresso;
- acompanhamento;
- API para integracoes futuras.
- base PT-BR de equipamentos, musculos, exercicios, alimentos e templates.

Regra simples:

- Admin configura.
- Professor acompanha.
- Aluno executa.

Comando tecnico de atualizacao da base PT-BR, usado apenas por suporte:

```bash
python3 manage.py seed-team-tupina-ptbr --username teamadmin
```

Esse comando atualiza equipamentos, musculos, exercicios, alimentos e templates publicos Team Tupina em portugues brasileiro.

## 14. Quando algo der errado

### Site nao abre

1. Testar no celular com 4G/5G.
2. Testar no computador.
3. Anotar o horario do erro.
4. Tirar print.
5. Chamar suporte tecnico.

### App nao abre

1. Acessar https://app.teamtupina.com.br/pt-br/user/login
2. Testar em guia anonima.
3. Verificar se aparece erro, tela branca ou login.
4. Tirar print.
5. Chamar suporte tecnico.

### Login nao funciona

1. Conferir se o usuario foi digitado exatamente.
2. Conferir se nao ha espaco antes/depois da senha.
3. Tentar em guia anonima.
4. Se falhar, pedir reset de senha.

### Aluno nao aparece para professor

1. Confirmar se o aluno existe.
2. Confirmar se o professor existe.
3. Confirmar se o aluno foi atribuido ao professor ou unidade correta.
4. Se nao encontrar a opcao, pedir suporte.

## 15. Como pedir suporte tecnico

Ao pedir suporte, envie sempre:

- O que voce tentou fazer.
- Qual link estava usando.
- Qual usuario estava logado.
- Print da tela.
- Horario aproximado.
- Se aconteceu no celular, computador ou ambos.

Modelo:

```text
Problema: nao consegui acessar o app.
Link: https://app.teamtupina.com.br/pt-br/user/login
Usuario: teamadmin
Horario: 14h20
Dispositivo: celular Android
O que apareceu: tela de erro / tela branca / senha invalida
Print: anexado
```

## 16. Regras de seguranca

- Nunca publicar senha em documento publico.
- Nunca enviar senha em grupo.
- Usar senha unica para cada sistema.
- Trocar senha ao trocar alguem da equipe.
- Nao criar admin root sem necessidade.
- Usar conta de professor para professor.
- Usar conta de aluno para aluno.
- Guardar acessos criticos em gerenciador de senhas.
